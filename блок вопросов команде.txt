Вопросы к Product Manager:

Бизнес-требования:

Какая ожидаемая нагрузка на API? Сколько запросов в минуту/час планируется?
Нужна ли аутентификация для доступа к API endpoints?
Требуется ли мониторинг и алертинг при недоступности внешнего API курсов?


Функциональные требования:

Нужно ли показывать статистику по курсам (мин/макс за день, средний курс)?
Требуется ли поддержка других валютных пар в будущем?
Какой формат timestamp предпочтителен для клиентов API?


Качество данных:

Какие допустимые отклонения курса для валидации данных от внешнего API?
Нужно ли логировать все запросы для аналитики бизнеса?
Требуется ли архивирование старых данных (какой период хранения)?


Интеграция:

Планируется ли интеграция с другими системами компании?
Нужен ли webhook для уведомления о значительных изменениях курса?



Вопросы к Team Lead:

Понимание задачи и архитектуры:

Правильно ли я понял требования? Нужно ли уточнить что-то в техническом задании?
Какую архитектуру вы рекомендуете для этого проекта? Стоит ли использовать Django Rest Framework или обычные Django views?
Есть ли в компании готовые паттерны для интеграции с внешними API, которые я должен использовать?


Выбор технических решений:

Какой внешний API для курсов валют лучше использовать? Есть ли корпоративные договоренности?
Где лучше хранить API ключи и настройки? В settings.py, переменных окружения или отдельном конфиге?
Какую базу данных использовать для этой задачи? SQLite для разработки достаточно?


Стандарты кода и код-ревью:

Какие есть code style guidelines в команде? Нужно ли настроить black/flake8?
Как организован процесс код-ревью? Кого назначить ревьюером?
Есть ли примеры похожих проектов в компании, на которые можно ориентироваться?


Тестирование и отладка:

Какой минимальный уровень покрытия тестами требуется для этой задачи?
Как лучше тестировать интеграцию с внешним API? Использовать моки или реальные запросы?
Где смотреть логи во время разработки и на проде?


Development Environment:

Нужно ли настраивать Docker для локальной разработки или можно работать в виртуальном окружении?
Какая версия Python и Django используется в проекте?
Есть ли готовый requirements.txt или нужно создавать с нуля?


Git workflow и деплой:

Какая git стратегия используется? Создавать feature branch или работать напрямую в develop?
Как происходит деплой? Нужно ли мне настраивать что-то для этого?
В какую среду деплоить для тестирования (staging/dev)?


Обработка ошибок и edge cases:

Что делать, если внешний API недоступен? Возвращать ошибку или последние сохраненные данные?
Нужно ли логировать все запросы к внешнему API для отладки?
Как обрабатывать ситуации, когда база данных недоступна?


Помощь и поддержка:

К кому обращаться, если застряну с техническими вопросами?
Есть ли документация по настройке окружения разработки?
Когда лучше показать промежуточные результаты для получения фидбека?


Временные рамки и приоритеты:

Какие части задачи наиболее критичны и должны быть реализованы в первую очередь?
Если не успею все за 2 дня, что можно отложить на следующую итерацию?
Нужно ли ежедневно отчитываться о прогрессе?


Production readiness:

Какие дополнительные требования нужно учесть для продакшена (мониторинг, алерты)?
Нужны ли migrations для продакшена или это пока только dev версия?
Требуется ли документация API для других разработчиков?
